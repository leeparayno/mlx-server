# Phase 4.2-4.4: Real MLX Batching Implementation Design

## Overview

**Goal:** Transform placeholder inference into real MLX model execution across three phases, achieving 6-7x throughput improvement through progressive optimization and testing.

**Approach:** Hybrid sequential implementation
- **Phase 4.2:** Real token generation with sequential forward passes (Week 1)
- **Phase 4.3:** KV cache optimization for memory efficiency (Week 2)
- **Phase 4.4:** Progressive load testing across model sizes (Week 3)

**Key Decision:** Start with sequential per-slot generation to validate architecture, then optimize with KV cache. No pluggable architecture needed - the method signature is the abstraction.

## Model Selection Strategy

Progressive testing with three model sizes:

1. **Small Model:** mlx-community/Qwen2.5-0.5B-Instruct-4bit (~340MB)
   - Fast iteration and bug finding
   - Quick downloads and testing cycles

2. **Medium Model:** mlx-community/gemma-2-2b-it-4bit (~1.5GB)
   - Performance benchmarking
   - Realistic workload characteristics

3. **Large Model:** mlx-community/Qwen2.5-7B-Instruct-4bit (~4GB)
   - Production-scale testing
   - Sustained load simulation
